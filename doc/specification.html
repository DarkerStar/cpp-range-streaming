General

This specification defines several functions and associated types that support the use of ranges with IOStreams.

Requirements

Range I/O functions are functions that take a range, possibly with other arguments, and return range I/O function objects that can be used in IOStreams insertion or extraction expressions to perform input or output using the range.

Range I/O functions and range I/O function objects must not copy the provided range. For ranges given as an lvalue, they must keep a reference or pointer to the range. Behaviour is undefined if the range I/O function object is used after the range goes out of scope. For ranges given as an rvalue, they must move the range into a locally-owned variable.

Range I/O function objects must be MoveConstructible, MoveAssignable, and Destructible. No standard range I/O function objects are default-constructable – they must be constructed by using the associated range I/O function.

The objects returned by range I/O functions should be usable with input or output streams (as applicable) either by lvalue reference or rvalue reference. This is to support the following two use cases (r is a Range): 

cin >> back_insert(r);

and 

auto b = back_insert(r);
cin >> b;
// b may now be queried for information about the last read

The objects returned by range I/O functions must have a non-static const member function named count, which returns a size_t value giving the number of elements that were read or written in the last I/O operation. The initial value is 0.

All input and output operations should set the stream width value to 0 immediately before returning.

Input requirements

Any formatting that would apply to the input of a single value type of the range must be applied to every element of the range.

After every element extraction, the state of the input stream is checked by the stream object’s conversion to bool, and if it is false, the operation ends immediately.

After the input operation, the range I/O function object’s count() function returns the number of values that were successfully read from the input stream, parsed, and stored in the range in that operation.

Output requirements

Any formatting that would apply to the output of a single value type of the range must be applied to every element of the range.

After every element insertion, the state of the output stream is checked by the stream object’s conversion to bool, and if it is false, the operation ends immediately.

After the output operation, the range I/O function object’s count() function returns the number of values that were successfully written to the output stream in that operation.

When the range is empty, an output operation on an ostream os should attempt an unformatted write of os.width() copies of os.fill() to os, then call os.width(0).

Class template range_back_inserter

#include <iosfwd>

namespace std {
  template <class Range>
  class range_back_inserter {
  public:
    size_t count() const noexcept;
    
  private:
    // Construction is only possible via back_insert()
    range_back_inserter(); // = delete;
  };
  
  template<class Range, class charT, class traits>
    basic_istream<charT, traits>&
      operator>>(basic_istream<charT, traits>&, range_back_inserter<Range>&);
  template<class Range, class charT, class traits>
    basic_istream<charT, traits>&
      operator>>(basic_istream<charT, traits>&, range_back_inserter<Range>&&);
  
  template<class Range>
    range_back_inserter<Range> back_insert(Range& range, RangeValueType v);
  template<class Range>
    range_back_inserter<Range> back_insert(Range& range);
  
  template<class Range>
    range_back_inserter<Range> back_insert_n(Range& range, size_t n, RangeValueType v);
  template<class Range>
    range_back_inserter<Range> back_insert_n(Range& range, size_t n);
}

range_back_inserter operations

range_back_inserter queries

size_t count() const noexcept;
Returns:
The number of elements read in the last input operation, or 0 if there have been no read operations.

range_back_inserter extractors

template<class Range, class charT, class traits>
  basic_istream<charT, traits>&
    operator>>(basic_istream<charT, traits>& is, range_back_inserter<Range>& p);

template<class Range, class charT, class traits>
  basic_istream<charT, traits>&
    operator>>(basic_istream<charT, traits>& is, range_back_inserter<Range>&& p);
Effects:
Acts as a RangeInputOperation, reading values of the range’s value type from is into the range referenced by p until bool(is) is false.
Each value v successfully read is added to the referenced range r by r.push_back(v).
Returns:
is.
Note:
Lvalue and rvalue overloads are required to handle the expected use cases, but both do the same thing.

back_insert

template <class Range>
  range_back_inserter<Range> back_insert(Range& range, RangeValueType v);
Returns:
A range_back_inserter<Range> object that holds a reference to range, set to read an unlimited number of values.
Note:
Input requires an extant object of the type being read in. This overload allows the user to provide an instance of a type that cannot be default-constructed, which will then be stored in the returned object and, on input, will be used as in cin >> v.
template <class Range>
  range_back_inserter<Range> back_insert(Range& range);
Note:
Same as the previous overload with a default-constructed RangeValueType.
